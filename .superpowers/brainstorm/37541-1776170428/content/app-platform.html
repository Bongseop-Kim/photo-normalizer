어떤 형태의 앱을 만들까요?

기술 스택과 사용 경험이 크게 달라집니다.

A

로컬 웹 앱

FastAPI 서버 + 브라우저 UI (React 또는 순수 HTML/JS)

장점

  • 기존 Python 코드 그대로 재사용
  • UI 자유도 최고
  • Before/After 슬라이더 등 기존 preview 기능 계승

단점

  • 브라우저에서 열어야 함 (앱 아이콘 없음)
  • 서버 별도 실행 필요
B

데스크톱 앱 (Electron / Tauri)

HTML/CSS/JS 프론트 + Python 백엔드를 subprocess로 호출

장점

  • 앱 아이콘, 트레이, 파인더 통합
  • Dock에 올릴 수 있음

단점

  • Node.js / Rust 빌드 환경 추가 필요
  • Python ↔ 앱 간 통신 설계 필요
  • 배포 복잡
C

Python GUI 앱 (PyQt6)

순수 Python 스택, QWidget 기반 네이티브 느낌 UI

장점

  • uv로 의존성 한 번에 관리
  • 기존 코드 직접 import
  • .app 번들 생성 가능 (PyInstaller)

단점

  • Before/After 슬라이더 같은 UI 구현이 번거로움
  • 웹 대비 UI 표현력 제한
